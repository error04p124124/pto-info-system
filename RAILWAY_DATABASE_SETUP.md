# Настройка базы данных PostgreSQL на Railway

## Проблема
На Railway не видно созданных пользователей, потому что используется пустая база данных.

## Решение

### Шаг 1: Добавьте PostgreSQL базу данных на Railway

1. Откройте ваш проект на Railway: https://railway.app
2. Выберите проект `pto-info-system`
3. Нажмите кнопку **"+ New"** → **"Database"** → **"Add PostgreSQL"**
4. Railway автоматически создаст базу данных и переменную окружения `DATABASE_URL`

### Шаг 2: Перезапустите деплой

После добавления PostgreSQL:
1. Railway автоматически перезапустит приложение
2. При запуске выполнится:
   - `python manage.py migrate` - создаст таблицы
   - `python manage.py demo_data_new` - **автоматически создаст пользователей!**
   - `python manage.py collectstatic` - соберет статику
   - Запустится gunicorn сервер

### Шаг 3: Проверьте

Откройте https://pto-info-system-production.up.railway.app/

**Тестовые аккаунты (создаются автоматически):**

| Логин | Пароль | Роль | Описание |
|-------|--------|------|----------|
| `admin` | `admin` | Суперпользователь | Полный доступ к админке |
| `head_pto` | `1234` | Начальник ПТО | Доступ ко всем скважинам и аналитике |
| `pto_engineer` | `1234` | Инженер ПТО | Доступ только к своим скважинам |

### Что было изменено в коде:

1. ✅ Добавлена поддержка PostgreSQL (`psycopg2-binary`, `dj-database-url`)
2. ✅ В [Procfile](Procfile) добавлена команда `demo_data_new` для автосоздания пользователей
3. ✅ В [settings.py](pto_info_system/settings.py#L113-L131) настроено автоопределение БД:
   - Если есть `DATABASE_URL` → PostgreSQL (production)
   - Если нет → SQLite (development)

### Альтернативный способ (если не хотите PostgreSQL)

Можно использовать SQLite на Railway (не рекомендуется):
1. Удалите PostgreSQL базу если создали
2. Загрузите ваш локальный `db.sqlite3` в Railway через GUI
3. Но **при каждом деплое база будет сбрасываться!**

PostgreSQL - правильное решение для production.
